# 学習支援アプリ 要件定義書

1. 概要
- 本アプリは中学生向けの学習支援Webアプリである。
- 問題演習を通じて利用者の理解度を分析し、苦手分野を重点的に学習できるよう支援する。
- 将来的には数学以外の教科（理科・英語など）や資格試験対策への拡張も想定する。

----
2. 目的
- 以下を実現する。
  - 問題演習による知識定着
  - 苦手分野の可視化
  - 苦手問題の重点学習
  - 解説表示による理解促進

---

3. 想定ユーザー
- 主ユーザー
  - 中学生
- 管理ユーザー
  - 保護者（初版では画面なし、将来対応）

---

4. システム構成
- 本システムは以下の構成とする。
- 学習アプリ（SPA）
  - 問題演習機能を提供する。
  - GitHub Pages にデプロイする静的サイト。
- 問題データ管理
  - 問題データ(JSON)をユーザーがローカルで管理し、アプリにインポートして使用する。
  - 問題データはリポジトリには含めない。

---

5. 技術スタック
   - フロントエンド
     - Vue 3 + Vite
     - Vuetify（UIコンポーネントライブラリ）
     - JavaScript（TypeScriptなし）
   - データ永続化
     - localStorage（バックエンドなし）
   - デプロイ
     - GitHub Pages

---

6. 対象環境
   - 対応デバイス
     - PC
     - スマートフォン
   - 利用ブラウザ
     - Chrome
     - Edge
     - Safari

7. 機能要件

7.1 問題出題
- ユーザーへ問題を出題する。
  - 要件
    - 1問ずつ表示する
    - 連続問題演習が可能
    - 5問モード
    - 10問モード
  - 将来拡張
    - 全問モード
    - 単元指定
    - 難易度指定
    - 苦手単元のみ

7.2 回答機能
  - ユーザーが回答を入力できる。
  - 要件
    - テキスト入力（数値・文字列）
    - 4択選択形式（問題データの `type` フィールドで指定）
    - 4択の選択肢順は毎回ランダムに並び替える
    - 数値答えの場合、誤答3つは自動生成（正答に近い値）

7.3 採点機能
  - 回答と正答を比較する。
  - 要件
    - 正誤判定
      - 全角・半角を統一して比較する
      - 前後のスペースを除去して比較する
    - 正答表示
    - 解説表示（JSONデータに事前記述した解説を表示）

7.4 学習履歴
- ユーザーごとの学習履歴をlocalStorageに保存する。
  - 保存内容
    - 問題ID
    - 回答内容
    - 正誤
    - 回答日時

7.5 苦手分析
  - ユーザーの理解度を分析する。
    - 問題単位
      - 出題回数
      - 正解回数
      - 正答率
    - 単元単位
      - 出題回数
      - 正解回数
      - 正答率

7.6 苦手問題優先出題
  - 理解度に応じて出題頻度を変更する。

  基本ルール
  - 重み = max(1 - 正答率, 0.1)
  - 未挑戦問題の重み = 1.0
  - 正答率100%でも最低重み0.1で出題される（完全除外しない）

  例
  - 正答率0%  → 重み1.0（最も出題されやすい）
  - 正答率50% → 重み0.5（通常）
  - 正答率100% → 重み0.1（最も出題されにくい）

7.7 問題インポート
  - ユーザーがローカルのJSONファイルを選択してアプリに取り込む。
  - 要件
    - トップ画面にインポートボタンを設置
    - 追加インポート対応（既存問題に追加する。上書きではない）
    - IDによる重複排除（既存IDはスキップ）
    - インポート後に「○問追加、○問スキップ（重複）」を表示

7.8 問題管理
  - インポート済み問題を管理できる。
  - 要件
    - 問題一覧表示
    - 単元（unit）でフィルタリング
    - チェックボックスで複数選択し一括削除
    - 全問削除ボタンは設けない（誤操作防止）

---

8. 画面構成

- トップ画面
  - 演習モード選択（5問 / 10問）
  - 問題データインポートボタン
  - 苦手分析・問題管理への導線
  - インポート済み問題数の表示

- 問題画面
  - 問題表示
  - 回答入力（テキスト or 4択）
  - 採点結果・解説表示
  - 次の問題へ進むボタン

- セッション結果画面
  - スコア表示
  - 不正解問題の解説一覧

- 苦手分析画面
  - 単元別正答率
  - 苦手問題一覧

- 問題管理画面
  - インポート済み問題の一覧・削除

## 演習フロー

トップ画面（問題数選択）→ 問題画面（1問ずつ回答）→ セッション結果画面（スコア＋不正解問題の解説）

---

9. 問題データモデル
   - 問題データはJSON形式で保持する。
   - サンプル
    <pre>
    {
      "id": "MATH001",
      "subject": "math",
      "grade": 2,
      "unit": "一次方程式",
      "tags": [
        "移項",
        "整数"
      ],
      "difficulty": 2,
      "type": "choice",
      "question": "2x + 3 = 9 を解け",
      "answer": "3",
      "explanation": "両辺から3を引いて 2x = 6、両辺を2で割って x = 3"
    }
    </pre>
   - `type`: `"choice"`（4択）または `"text"`（テキスト入力）

---

10. 教科拡張性
   - 初版対象
     - 数学
   - 将来対象
     - 理科
     - 英語
     - 社会
     - 国語
     - 資格試験問題

    ※問題データ構造は教科依存としない。

---

11. 非機能要件
- 性能
  - 問題表示3秒以内
- 保守性
  - 問題データとアプリ本体を分離する
- 拡張性
  - 教科追加可能
  - 問題形式追加可能

---

12. 初版(MVP)スコープ
以下のみ実装対象とする。
- 必須
  - 問題出題（5問 / 10問モード）
  - 回答入力（テキスト / 4択）
  - 採点
  - 学習履歴保存
  - 苦手問題優先出題
  - 問題インポート（JSONファイルアップロード）
  - 問題管理（一覧・削除）
  - 苦手分析画面
- 将来実装
  - AI解説・類題生成
  - 全問モード
  - 保護者向け画面
  - 問題作成アプリ（PDF読み込み→問題生成→JSON出力）
  - 忘却曲線対応
  - OCR連携
  - 問題管理画面（将来の高機能版）

以上。
